# Портфолио бизнес-аналитика - кейс «Конвертация валюты»

Демонстрационное портфолио для стажировки бизнес-аналитиком. Это не набор разрозненных работ, а один сквозной кейс вокруг продукта конвертации валюты в банке: от понимания рынка и моделирования процессов до требований, продуктовой документации и метрик на SQL. Все данные вымышленные, кейс подготовлен на основе открытых источников.

## О кейсе

Продукт - обмен валюты для розничных клиентов в двух каналах: в мобильном приложении и наличными в отделении. Кейс показывает один продукт с разных сторон: как устроен валютный рынок и экономика конвертации, как течёт процесс, какие к нему требования, как это описано для команды и как измеряется результат.

## Состав портфолио

### 1. Бизнес-процессы (BPMN 2.0, Bizagi)
- **Конвертация в мобильном приложении** - сквозной процесс с пулами и дорожками, обменом сообщениями с клиентом и внешним источником курса, каскадом контролей (лимит, AML, средства), таймером устаревания котировки и откатом при сбое зачисления.
  [Схема (PNG)](diagrams/conversion_app.png) · [Исходник (.bpmn)](diagrams/conversion_app.bpmn)
- **Наличная конвертация в отделении** - процесс с очной идентификацией, проверкой подлинности купюр, пороговым контролем ПОД/ФТ и выдачей наличных. Пара для сравнения каналов.
  [Схема (PNG)](diagrams/conversion_branch.png) · [Исходник (.bpmn)](diagrams/conversion_branch.bpmn)

### 2. Требования
- **User stories с критериями приёмки** - требования, «нарезанные» из процесса, в формате Given/When/Then, с приоритетами по MoSCoW. [Открыть](requirements/user_stories.md)
- **Мини-ТЗ / SRS** - детальная спецификация одной фичи (показ курса и контроль актуальности котировки): функциональные и нефункциональные требования, бизнес-правила, сценарии. [Открыть](requirements/srs_pokaz_kursa.md)

### 3. Продуктовая документация
- **Продуктовая страница** - описание продукта, участники, бизнес-правила, точки контроля, экономика и метрики (формат продуктовой страницы Confluence). [Открыть](docs/product_page.md)

### 4. Данные и метрики
- **SQL-витрина операций** - схема витрины и запросы под ключевые продуктовые метрики: доход по спреду, конверсия, структура отказов, время до подтверждения. [Открыть](sql/vitrina_konvertacii.sql)

## Как читать кейс

Артефакты связаны в логическую цепочку: BPMN-схемы описывают процесс, user stories и SRS превращают процесс в требования, продуктовая страница собирает контекст для команды, SQL-витрина измеряет результат. Каждый артефакт ссылается на соседние.

## Инструменты

Bizagi Modeler (BPMN 2.0), Markdown / Confluence, SQL (PostgreSQL).
