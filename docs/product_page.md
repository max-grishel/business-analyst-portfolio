## 1. Кратко о продукте

Продукт позволяет клиенту банка обменять одну валюту на другую прямо в мобильном приложении, без визита в отделение. Клиент выбирает валютную пару и сумму, видит итоговый курс с учётом спреда банка, подтверждает сделку - после чего средства списываются в валюте-источнике и зачисляются в валюте-назначения.

Для кого: розничные клиенты банка, у которых есть счета в соответствующих валютах.

Ценность для клиента: быстрый обмен в любое время без посещения отделения, прозрачный курс до подтверждения.

Ценность для банка: комиссионный доход за счёт спреда, рост вовлечённости в мобильном приложении.

## 2. Ключевые термины

| Термин | Определение |
|---|---|
| Валюта-источник | Валюта, которая списывается со счёта клиента (откуда конвертируем) |
| Валюта-назначения | Валюта, которая зачисляется клиенту (куда конвертируем) |
| Рыночный курс | Курс, получаемый от внешнего источника котировок |
| Спред | Наценка банка к рыночному курсу; основной источник дохода продукта |
| Котировка | Итоговый курс и сумма, показанные клиенту; имеет срок жизни (TTL) |
| TTL котировки | Время, в течение которого показанный курс действителен (например, 60 секунд) |
| AML / ПОД-ФТ | Проверка операции на соответствие требованиям по противодействию отмыванию доходов (115-ФЗ) |

## 3. Как работает продукт

Процесс от заявки клиента до завершения операции описан на отдельной схеме BPMN.

> ![Схема процесса конвертации в приложении](C:\Users\DADDY\Desktop\стаж\diagrams\conversion_app.png)

Основной сценарий по шагам:

1. Клиент инициирует конвертацию: выбирает валютную пару и сумму.
2. Приложение запрашивает рыночный курс у внешнего источника.
3. Банк рассчитывает итоговый курс с учётом спреда и формирует котировку.
4. Клиенту показывается курс и итоговая сумма.
5. Клиент подтверждает сделку (пока действует TTL котировки).
6. Банк последовательно выполняет контроли: лимит → AML → достаточность средств.
7. При успешном прохождении: списание валюты-источника, затем зачисление валюты-назначения.
8. Клиент получает уведомление о выполненной конвертации.

## 4. Участники процесса

| Участник | Зона ответственности |
|---|---|
| Клиент | Инициирует конвертацию, подтверждает сделку |
| Мобильное приложение (фронт) | Принимает заявку, показывает курс, передаёт подтверждение |
| Процессинг и казначейство банка | Расчёт курса, контроли, списание и зачисление |
| Внешний источник рыночного курса | Предоставляет рыночную котировку |

## 5. Бизнес-правила

- Курс показывается клиенту с учётом спреда банка; спред задаётся для каждой валютной пары.
- Котировка действительна ограниченное время (TTL, например 60 секунд). По истечении срока курс пересчитывается, и клиенту показывается новая котировка.
- Подтверждение по устаревшей котировке не принимается - сделка возможна только по актуальной цене.
- Перед исполнением операция проходит контроль лимитов, AML-проверку и проверку достаточности средств.
- Списание валюты-источника выполняется до зачисления валюты-назначения.

## 6. Точки контроля и обработка исключений

| Контроль / событие | Если не пройдено | Результат для клиента |
|---|---|---|
| Лимит на операцию | Отклонение | «Отклонено по лимиту» |
| AML-проверка | Отклонение / передача на ручную проверку | «Отклонено комплаенсом» |
| Достаточность средств | Отклонение | «Недостаточно средств» |
| TTL котировки истёк | Возврат на пересчёт курса | Показывается новая котировка |
| Сбой зачисления после списания | Откат: возврат списанной суммы | «Сбой, средства возвращены» |

Ключевой риск продукта: списание прошло, а зачисление сорвалось. Поэтому на шаге зачисления предусмотрен откат, гарантирующий, что клиент не останется без средств с обеих сторон.

## 7. Экономика продукта

Доход формируется за счёт спреда - разницы между рыночным курсом и курсом, по которому проводится операция для клиента. Размер спреда зависит от валютной пары, объёма операции и канала.

## 8. Отличие от наличной конвертации в отделении

| Параметр | В приложении | В отделении (наличная) |
|---|---|---|
| Канал | Мобильное приложение | Касса отделения |
| Курс | Онлайн-котировка, обычно уже спред | Курс отделения, как правило шире спред |
| Идентификация | Уже выполнена при входе в приложение | Очно, по документу |
| Скорость | Мгновенно | Зависит от очереди |

## 9. Метрики продукта

- Конверсия из открытия экрана конвертации в завершённую операцию.
- Доля отказов с разбивкой по причинам (лимит / AML / средства / сбой).
- Доля котировок, истёкших по TTL (косвенно - удобство и скорость экрана подтверждения).
- Среднее время от показа курса до подтверждения.
- Средний объём операции и доход по спреду.
